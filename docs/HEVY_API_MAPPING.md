## Important: What Hevy actually provides

The Hevy **public REST API** does **not** have separate endpoints for:
- `/exercises`, `/muscles`, `/equipment`, `/categories`
- `/images`, `/media`, exercise instructions, or videos

Exercise catalog data comes from **`GET /v1/exercise_templates`** only.

### Live sample exercise template fields

- `id`: string e.g. "3BC06AD3"
- `title`: string e.g. "21s Bicep Curl"
- `type`: string e.g. "weight_reps"
- `primary_muscle_group`: string e.g. "biceps"
- `secondary_muscle_groups`: array<undefined> (length 0)
- `equipment`: string e.g. "barbell"
- `is_custom`: boolean

## Endpoint inventory

| Endpoint | Purpose |
|----------|---------|
| `GET /v1/user/info` | Authenticated user profile |

## Schema gaps (Hevy does NOT provide — optional future fields)

These were requested in the import plan but are **not in Hevy's API response**:

- Difficulty level (beginner/intermediate/advanced)
- Force type (push/pull/static)
- Mechanic (compound/isolation)
- Multiple image URLs / video URLs
- Alternative names / tags / popularity score

**Recommendation:** Keep using Wger (`pnpm db:import`) for images and instructions. Use Hevy import for exercise **names**, **muscle/equipment taxonomy**, and **type** alignment.
